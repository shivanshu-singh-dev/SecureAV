# Layer 2 — Synthetic Sensor Stream Detection

Detects whether incoming camera/LiDAR data is genuine or has been
replaced/faked, using two independent checks:

1. **Hardware fingerprinting** — a trained classifier looks at a frame's
   noise pattern (real sensors are "dirty" in specific physical ways;
   synthetic/replayed streams are too clean or have the wrong noise
   signature).
2. **Cross-modal consistency** — a rule-based check confirms camera and
   LiDAR agree on what physically exists in each region. They can't
   legitimately disagree about a solid object's presence.

Both scores are combined into a single `alert_score` (0.0 = trustworthy,
1.0 = high confidence attack) for the team's Dempster-Shafer fusion step.

## Files

| File | Purpose |
|---|---|
| `features.py` | Extracts the noise-fingerprint feature vector from a frame; also frame-hashing for replay detection |
| `cross_modal.py` | Rule-based camera/LiDAR agreement check |
| `model_train.py` | Trains the fingerprint classifier, saves `model.pkl` |
| `detector.py` | `Layer2Detector` — the public integration class |
| `test_layer2.py` | Scenario tests (genuine / synthetic / mismatch) |
| `model.pkl` | Trained classifier (included, generated by `model_train.py`) |
| `generate_camera_dataset.py` | Generates `data/real/` and `data/synthetic/` |
| `generate_lidar_dataset.py` | Generates `data/lidar_cross_modal_dataset.json` |
| `data/` | Included generated datasets (see Dataset section below) |

## Training

Put your data here before training:

```
data/
    real/         <- genuine camera frames (.png/.jpg), from CARLA or your simulator
    synthetic/    <- attack frames: blurred/denoised copies of real frames,
                     or replayed/out-of-order frames
```

Then run:

```bash
python model_train.py
```

This prints test accuracy and writes `model.pkl`.

## Integration

```python
from modules.layer2_sensor.detector import Layer2Detector

detector = Layer2Detector()          # call once at startup, loads model.pkl

# call every cycle:
result = detector.detect(camera_frame, regions=regions)
```

### Input

- `camera_frame`: numpy array (BGR or grayscale), the current camera frame.
- `regions` (optional): list of dicts, one per region of interest, e.g.:

```python
regions = [
    {"region_id": "front_center", "camera_object_present": True, "lidar_object_present": True},
    {"region_id": "front_left",   "camera_object_present": False, "lidar_object_present": False},
]
```

If `regions` is omitted, only the fingerprint check runs that cycle and
`alert_score` is based on `fingerprint_score` alone.

### Output

```python
{
    "synthetic_flag": bool,        # True = fingerprint suggests fake/tampered stream
    "fingerprint_score": float,    # 0.0 (genuine) - 1.0 (synthetic)
    "cross_modal_mismatch": bool,  # True = camera/LiDAR disagree on some region
    "consistency_score": float,    # 0.0 (full disagreement) - 1.0 (full agreement)
    "alert_score": float,          # 0.0-1.0 combined score -> feed into Dempster-Shafer
}
```

## Dataset

Two datasets are included, ready to train/validate against immediately —
useful for getting Layer 2 working end-to-end before real CARLA capture is
available, and for stress-testing the detection logic with controlled,
class-balanced data.

### Camera dataset (`data/real/`, `data/synthetic/`)

Generated by `generate_camera_dataset.py`. 400 genuine frames + 400 attack
frames by default (configurable via `--samples`). Each genuine frame has a
procedurally varied scene (gradient direction, road band, object blobs,
brightness/contrast jitter) plus physically-motivated sensor noise (Gaussian
thermal noise + sparse hot pixels, both with randomized intensity per
frame). Variety matters more here than photorealism, since the classifier
learns noise statistics, not scene content — without variety it would
memorize one scene instead of the underlying physical signal.

Attack frames are split across three types (rotated for class balance):

| Attack type | What it simulates | Caught by |
|---|---|---|
| `denoised` | Blurred/smoothed real frame ("too clean") | Fingerprint classifier |
| `smoothed` | Aggressive downscale/upscale (low-effort AI regen) | Fingerprint classifier |
| `replayed` | A genuine past frame reused verbatim as if live | Frame-hash replay detector |

Regenerate or resize anytime:
```bash
python generate_camera_dataset.py --samples 600 --height 160 --width 240
```

### LiDAR / cross-modal dataset (`data/lidar_cross_modal_dataset.json`)

Generated by `generate_lidar_dataset.py`. 400 consistent samples + 400
attack samples, each describing one frame's camera-vs-LiDAR object presence
across 6 regions (front-left/center/right, rear-left/center/right). Attack
samples flip exactly one region to simulate either a `camera_blind_spot`
(LiDAR sees an object, camera was tampered to hide it) or
`phantom_camera_object` (camera reports an object, LiDAR sees nothing).

```bash
python generate_lidar_dataset.py --samples 600
```

### Validated performance (on the included generated datasets)

| Check | Method | Accuracy |
|---|---|---|
| Cross-modal consistency (`cross_modal.py`) | Rule-based | **100%** (800/800) — exact by construction, not a trained estimate |
| Fingerprint classifier — denoised attacks | Trained classifier | **100%** (134/134) |
| Fingerprint classifier — smoothed attacks | Trained classifier | **100%** (133/133) |
| Replay attacks | Frame-hash history (`detector.py`) | Caught when frame repeats within `REPLAY_HISTORY_SIZE` (300-frame) window — **not** detectable via noise statistics, by design |

**Important honest note on replay attacks:** a replayed frame is a genuine,
correctly-noised real frame reused verbatim — its noise statistics are
*identical* to real sensor data by definition. The fingerprint classifier
will never catch this (and `replayed` frames are deliberately excluded from
its training set in `model_train.py` to avoid injecting label noise). It's
caught instead by `Layer2Detector`'s separate frame-hash history check,
which flags any frame matching one seen in the last ~300 cycles. This is
two genuinely different detection mechanisms for two genuinely different
attack shapes — worth knowing for Q&A, since "100% on everything" would be
the wrong (and less defensible) claim.
